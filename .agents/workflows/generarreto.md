Objetivo: Crear un ejercicio a medida para poner a prueba lo aprendido.

Descripción: Ejecuta la fase 3 del Bucle Pedagógico. Diseña el desafío para tu proyecto personal.

Instrucciones del flujo:

1. Leer el concepto validado en la clase más reciente.

2. Diseñar un requerimiento técnico nuevo que deba ser implementado dentro de la arquitectura del paquete pildorasinformatica_gemini.

3. El requerimiento debe ser sobre el proyecto que ya existe, es una forma de mejorar, refactorizar, ampliar ese proyecto de practica, para simular un proyecto profecional que crese con el tiempo.

4. Entregar el requerimiento en formato de "Checklist de Criterios de Aceptación".

5. Recordar la Regla de Oro: no incluir nada de código en la descripción del reto.

6. Validar estrictamente: convenciones de nombres (PascalCase para clases, camelCase para métodos/variables), uso de modificadores de acceso (encapsulamiento) y correctas anotaciones como @Override, uso de convenciones en el nombre de las varialbles o metodos.
